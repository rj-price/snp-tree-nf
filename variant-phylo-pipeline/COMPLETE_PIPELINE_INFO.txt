                              📋 PIPELINE OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

PURPOSE:
Complete bioinformatics pipeline for phylogenetic analysis from NGS reads,
integrating quality control, variant calling, and phylogenetic tree construction.

INPUT:
- Paired-end FASTQ reads (Illumina)
- Reference genome (FASTA)
- Samplesheet (CSV format)

OUTPUT:
- Quality reports (FastQC + MultiQC)
- Aligned BAM files with statistics
- Filtered VCF files (SNPs only)
- Phylogenetic tree (Newick format)
- Alignment matrices (PHYLIP + NEXUS)

═══════════════════════════════════════════════════════════════════════════════
                            🔬 SCIENTIFIC WORKFLOW
═══════════════════════════════════════════════════════════════════════════════

STAGE 1: Quality Control & Preprocessing
├─ [1] Samplesheet validation (Python)
├─ [2] FastQC on raw reads (Andrews 2010)
├─ [3] Trimmomatic adapter/quality trimming (Bolger 2014)
└─ [4] FastQC on trimmed reads

STAGE 2: Read Alignment
├─ [5] BWA index reference genome (Li & Durbin 2009)
├─ [6] BWA-MEM alignment with read groups (Li 2013)
├─ [7] SAMtools SAM→BAM conversion (Li et al. 2009)
├─ [8] Coordinate sorting and indexing
└─ [9] Alignment statistics

STAGE 3: Variant Discovery
├─ [10] SAMtools faidx reference indexing
├─ [11] BCFtools mpileup variant calling (Danecek et al. 2021)
└─ [12] SNP filtering (QUAL≥20, DP≥10, MQ≥30)

STAGE 4: Phylogenetic Analysis
├─ [13] VCF to PHYLIP conversion (Python)
├─ [14] RAxML ML tree (GTRGAMMA, 100 BS) (Stamatakis 2014)
└─ [15] VCF to NEXUS conversion (for SplitsTree)

STAGE 5: Quality Reporting
└─ [16] MultiQC aggregated report (Ewels et al. 2016)

BASIC EXECUTION:
```

    --input samples.csv \
    --fasta reference.fa \
    --outdir results

WITH OPTIONS:
```

    --input samples.csv \
    --fasta reference.fa \
    --adapters TruSeq3-PE.fa \
    --outdir results \
    --max_cpus 16 \
    --max_memory 32.GB

RESUME FAILED RUN:
```
nextflow run main.nf \
    --input samples.csv \
    --fasta reference.fa \
    -resume
```

                           🎯 KEY PARAMETERS
═══════════════════════════════════════════════════════════════════════════════

REQUIRED:
--input         Path to samplesheet CSV (sample_id, read1, read2)
--fasta         Path to reference genome FASTA

OPTIONAL:
--outdir        Output directory [default: results]
--adapters      Adapter sequences for trimming [default: none]
--max_cpus      Maximum CPUs [default: 8]
--max_memory    Maximum memory [default: 16.GB]
--max_time      Maximum time [default: 48.h]

TRIMMOMATIC PARAMETERS:
--trim_quality          Quality threshold [default: 20]
--trim_min_length       Minimum read length [default: 36]
--trim_window_size      Sliding window size [default: 4]
--trim_headcrop         Bases to crop from start [default: 10]

VARIANT FILTER PARAMETERS:
--filter_qual           Minimum quality [default: 20]
--filter_depth          Minimum depth [default: 10]
--filter_mq             Minimum mapping quality [default: 30]

RAXML PARAMETERS:
--raxml_model           Evolution model [default: GTRGAMMA]
--raxml_bootstraps      Bootstrap replicates [default: 100]

COMMON ISSUES:

1. SAMPLESHEET ERRORS
   - Check CSV format (headers required)
   - Verify file paths are correct
   - Look for duplicate sample IDs

2. MEMORY ERRORS
   - Increase max_memory parameter
   - Adjust per-process memory in base.config
   - Enable retry with more memory

3. PROCESS FAILURES
   - Check .nextflow.log for details
   - Verify software installations
   - Use -resume to restart from checkpoint

4. ALIGNMENT ISSUES
   - Ensure reference is uncompressed
   - Check read group format
   - Verify BWA index completion

5. VARIANT CALLING PROBLEMS
   - Check BAM file quality
   - Verify reference indexing
   - Review filtering thresholds
